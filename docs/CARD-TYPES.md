# Card-type rules & constraints

How each card type behaves in the engine, per the official Riftbound rules.
Sources: official Core Rules / FAQ, riftbound.gg, fextralife wiki (Token),
randomcrit rules 100-183.

## Types
| Type | In play? | Exhausts? | On resolve/leave | Notes |
|------|----------|-----------|------------------|-------|
| **Unit** | Yes (Base / Battlefields) | Yes | Trash when defeated | Your fighters; enter exhausted (Accelerate → ready). |
| **Spell** | No | **Never** | **Straight to Trash** after its effect | One-shot. Not a permanent. ✅ enforced: spells resolve then trash; can't be exhausted. |
| **Gear** | Yes (Base) | Yes | Trash | Ongoing; attaches to a unit, can grant +Might. |
| **Rune** | Base / Rune Pool | Yes (for Energy) | Recycled to rune deck | Separate 12-card Rune deck; never in the main deck. |
| **Battlefield** | Center row | No | — | Placed at setup; the scoring objectives. |
| **Legend** | Legend Zone | **Only via its own ability** | Can't leave the zone | Persistent; readies in Awaken. ✅ enforced: a Legend exhausts only through `ACTIVATE_LEGEND` (no free tapping, no enemy exhaust). |

## Supertypes (cross-type categories)
- **Token** (e.g. **Recruit**): generated by effects, **never in a deck or
  booster**. Lives in a separate **token pile** and is created onto the Base by
  a token-making effect. ✅ enforced:
  - Excluded from the deck builder pool and flagged in deck validation.
  - Stripped from any deck at match setup (defensive).
  - Summoned via the `CREATE_TOKEN` action from `PlayerState.tokenPile` (the
    baseline Recruit is available to every player). In the real game the pull is
    gated by specific card effects; until per-card effects are fully scripted,
    the board exposes a "+ Recruit" button to represent that generation.
  - Recruit = a 0-cost, 1-Might token unit with no keywords.
- **Champion** (units only): a unit tied to a Legend; the Chosen Champion is set
  aside in the Champion Zone and is always playable from there. Deckbuilding:
  the starting champion copy counts toward the 3-of limit.
- **Signature** (any type): deckbuilding-restricted cards tied to a Legend.
- **Basic** (basic Runes): the plain domain runes.

## What's enforced now
- Spells: resolve → Trash, never exhausted, never sit on the board.
- Legends: exhaust only through their ability; ready in Awaken; can't be decked.
- Tokens/Recruits: out of decks, in a separate pile, summoned (not drawn).
- Runes/Battlefields: already separated (rune deck / battlefield slots).

## Still simplified / TODO
- **Champion/Signature deckbuilding limits** aren't fully validated (e.g.
  Signature copy caps); only the token exclusion + Champion Zone exist.
- Token generation is a manual button (real game = specific card effects).
- Per-card effects that create *other* tokens (Buff, etc.) aren't scripted.
