# Handler-coverage analysis вЂ” where to apply our learnings

> **What this is.** An objective, pool-wide map of which cards the engine already
> resolves, which it punts to `manual`, and вЂ” for the manual ones вЂ” which mechanic
> family they belong to, so we know exactly which cards a given handler would unlock.
> Generated by running the real engine parsers (`cardIntent.prefillSpecFromCard` в†’
> `effects.ts`/`keywords.ts`/`triggers.ts`) over all **984** cards. Raw per-card data:
> [`handler-coverage.data.json`](handler-coverage.data.json). Pairs with the meta-ranked
> [`gap-matrix.md`](gap-matrix.md) (impact = how many of the 10 meta decks need a mechanic)
> and the slot grammar in [`card-grammar.md`](card-grammar.md).

## Method & caveats
- For each card we record: the `ParsedEffect` mechanic keys the parser detects, the
  keywords, the triggers, a `manual` flag (the parser couldn't resolve some text), and
  whether the card is **hand-coded** (a bespoke per-card branch).
- `handCoded` is detected by the card's bare name appearing as a quoted literal in
  `engine.ts`/`battlefieldScripts.ts`. This catches name-keyed handlers (в‰€99 cards) but
  **misses the few regex-keyed ones** (Lee Sin - Centered, Garen - Commander) вЂ” treat the
  hand-coded count as a floor.
- A card is effectively **covered** if it's `fully-parsed`, `keyword-only`, `vanilla`, or
  `handCoded`. The genuine work surface is **`manual`/`partial` AND not hand-coded в‰€ 275 cards.**

## Coverage summary (984 cards)
| Bucket | Count | % | Meaning |
|---|--:|--:|---|
| `fully-parsed` | 490 | 50% | every text clause maps to a `ParsedEffect` key |
| `keyword-only` | 109 | 11% | only `[Keyword]`s (Tank/Assault/вЂ¦), already handled |
| `manual-unhandled` | 332 | 34% | text present, parser extracted nothing usable |
| `partial` | 29 | 3% | some clauses parsed, some flagged `manual` |
| `vanilla` | 24 | 2% | no rules text (basic runes etc.) |
| hand-coded | 99 | вЂ” | bespoke per-card/ -battlefield handlers (overlaps the above) |

So **~63% of the pool resolves on its own** (parsed + keyword + vanilla), bespoke code covers
more of the rest, leaving **~290 cards** (manual/partial and not hand-coded) that need a
handler вЂ” clustered into the families below.

> **Important nuance (discovered this pass): the scan measures PARSE coverage, not RESOLUTION.**
> A card can be `fully-parsed` (the parser understood the text) yet not be *resolved* by the
> reducer. This matters most for **activated abilities** вЂ” see below.

## Activated abilities вЂ” the real story (updated)
Activated abilities (`<cost>: <effect>`) are **already detected and offered generically**
(`unitActivatedAbility` + `ACTIVATE_UNIT`); **167 cards** have one. `ACTIVATE_UNIT` resolves a
growing inline set of effects: damage, tempMight, buff, move-to-base, return-to-hand, draw,
channel, recruits, named/gold tokens, predict, gainXP, **`[Add]` resources** (rune-ramp: the
Seals, Energy Conduit, Malzahar - Fanatic вЂ” add Power/Energy to the pool), and вЂ” added in the
#1-continuation pass вЂ” **stun, kill, grant-Assault/Ganking, ready (a unit / your units / self /
runes / others-here), and channel-exhausted**. So the bulk of the formerly-stuck activated
cards (the ready/stun/kill/grant/channel-exhausted family) now actually resolve; targeting for
the new targeted effects (stun/kill/grant/ready-unit) is wired in both MatchPage and OnlinePage.
**Still unresolved** (parse but don't yet resolve via activation): **score, move-to-battlefield
(move an enemy/unit to a battlefield), dealMight, copy.** These are the remaining activated tail
and overlap with the gap-matrix mechanics ranked below. Known limitation: `[Add]`/Seal abilities
are printed at **[Reaction] speed** but `ACTIVATE_UNIT` still gates on `requireActiveAction`
(your turn only), so a Seal can't be tapped mid-chain / on the opponent's turn yet.

## Hand-coded в†’ generalizable
66 cards are hand-coded yet still parser-`manual` (correctly вЂ” they need bespoke logic). A few
are worth **generalizing into a reusable handler** because the pool has many siblings:
- **"deal damage equal to its/my Might"** вЂ” Yasuo - Remorseful, Yone - Blademaster, Warwick are
  bespoke; в‰Ґ15 cards share the phrasing в†’ a `dealMight` effect primitive.
- **Reflection / copy** вЂ” Mirror Image, LeBlanc, Keeper of Masks each bespoke в†’ one `copy` effect.
- **stat-scaling Might** (Draven points / Mundo trash) в†’ a small `statScale(source)` primitive.
- Most other hand-coded cards are genuine one-offs; leave them.

## Using / regenerating
The dataset was produced by a throwaway env-gated generator (`GEN_COVERAGE=1 npx vitest run`
over a temp file importing `cardIntent` + `CARDS`); it's not part of the standing suite. Filter
[`handler-coverage.data.json`](handler-coverage.data.json) by `bucket` / `mechanics` /
`handCoded` to get the exact card list for any fix. Implementing these (the **fix campaign**) is
the next, separate effort вЂ” start at #1 (activated abilities) for the most cards-per-hour.
