# Handler-coverage analysis ‚ÄĒ where to apply our learnings

> **What this is.** An objective, pool-wide map of which cards the engine already
> resolves, which it punts to `manual`, and ‚ÄĒ for the manual ones ‚ÄĒ which mechanic
> family they belong to, so we know exactly which cards a given handler would unlock.
> Generated by running the real engine parsers (`cardIntent.prefillSpecFromCard` ‚Üí
> `effects.ts`/`keywords.ts`/`triggers.ts`) over all **984** cards. Raw per-card data:
> [`handler-coverage.data.json`](handler-coverage.data.json). Pairs with the meta-ranked
> [`gap-matrix.md`](gap-matrix.md) (impact = how many of the 10 meta decks need a mechanic)
> and the slot grammar in [`card-grammar.md`](card-grammar.md).

## Method & caveats
- For each card we record: the `ParsedEffect` mechanic keys the parser detects, the
  keywords, the triggers, a `manual` flag (the parser couldn't resolve some text), and
  whether the card is **hand-coded** (a bespoke per-card branch).
- `handCoded` is detected by the card's bare name appearing as a quoted literal in
  `engine.ts`/`battlefieldScripts.ts`. This catches name-keyed handlers (‚Čą99 cards) but
  **misses the few regex-keyed ones** (Lee Sin - Centered, Garen - Commander) ‚ÄĒ treat the
  hand-coded count as a floor.
- A card is effectively **covered** if it's `fully-parsed`, `keyword-only`, `vanilla`, or
  `handCoded`. The genuine work surface is **`manual`/`partial` AND not hand-coded ‚Čą 275 cards.**

## Coverage summary (984 cards)
| Bucket | Count | % | Meaning |
|---|--:|--:|---|
| `fully-parsed` | 406 | 41% | every text clause maps to a `ParsedEffect` key |
| `keyword-only` | 213 | 22% | only `[Keyword]`s (Tank/Assault/‚Ä¶), already handled |
| `manual-unhandled` | 315 | 32% | text present, parser extracted nothing usable |
| `partial` | 26 | 3% | some clauses parsed, some flagged `manual` |
| `vanilla` | 24 | 2% | no rules text (basic runes etc.) |
| hand-coded | 99 | ‚ÄĒ | bespoke per-card/ -battlefield handlers (overlaps the above) |

So **~63% of the pool already resolves** (parsed + keyword + vanilla), bespoke code covers
~66 more of the manual ones, leaving **~275 cards** that need a handler ‚ÄĒ and they cluster
tightly into a handful of families below.

## The headline lever: wire generic **activated abilities** (`<cost>: <effect>`)
**~75 manual/partial cards are activated abilities** ‚ÄĒ `:rb_exhaust:`/`:rb_energy_N:`/rune
`: <effect>`. The *effect* almost always already parses (buff / ready / deal / draw / move /
token); what's missing is auto-detecting the **activated KIND** (`card-grammar.md` ¬ß3) and
offering it via `ACTIVATE_UNIT`. Examples: Iron Ballista, Unlicensed Armory, Caitlyn -
Patrolling, Arena Bar, Orb of Regret, Ravenborn Tome, Garbage Grabber. **One generic
"parse `<cost>: <effect>` ‚Üí offer as an activated ability" pass would reclaim the single
largest slice of the manual bucket.**

## Hand-coded ‚Üí generalizable
66 cards are hand-coded yet still parser-`manual` (correctly ‚ÄĒ they need bespoke logic). A few
are worth **generalizing into a reusable handler** because the pool has many siblings:
- **"deal damage equal to its/my Might"** ‚ÄĒ Yasuo - Remorseful, Yone - Blademaster, Warwick are
  bespoke; ‚Č•15 cards share the phrasing ‚Üí a `dealMight` effect primitive.
- **Reflection / copy** ‚ÄĒ Mirror Image, LeBlanc, Keeper of Masks each bespoke ‚Üí one `copy` effect.
- **stat-scaling Might** (Draven points / Mundo trash) ‚Üí a small `statScale(source)` primitive.
- Most other hand-coded cards are genuine one-offs; leave them.

## Prioritized fix queue (pool reach √ó meta impact)
1. **Generic activated abilities** (`<cost>: <effect>`) ‚ÄĒ **~75 cards**, biggest single pool
   lever; the effects already parse. (`card-grammar.md` ¬ß3 KIND `activated`.)
2. **Play-from-deck/trash for free** ‚ÄĒ gap-matrix **#1** (5 meta decks) + the trash-recursion
   family; a fully-missing `playFrom` effect.
3. **Replacement / "would die" layer** (heal / recall / prevent) ‚ÄĒ gap-matrix #6; the only
   fully-missing KIND; unlocks Sett, Zhonya's, Guardian Angel + ~16 cards.
4. **Stun triggers + "if target stunned"** ‚ÄĒ gap-matrix #3 (4 decks).
5. **Move an enemy unit to a chosen battlefield** ‚ÄĒ gap-matrix #7; extend `moveUnit` past the
   Charm case to the move family's enemy/with variants.
6. **Direct `score(N)` + `on.winCombat`** ‚ÄĒ gap-matrix #8.
7. **Enemy-death triggers** (`on.death.enemy`) ‚ÄĒ gap-matrix #5.
8. **`dealMight` + `copy` primitives** ‚ÄĒ generalize the bespoke handlers above (~20 cards).
9. **Counter "unless pay N"** + conditional/aura cost-shaping ‚ÄĒ Hard Bargain, Vex, ~19 aura cards.
10. **`discard` effect + reveal-hand** ‚ÄĒ Mindsplitter family + on-discard trigger event.

## Using / regenerating
The dataset was produced by a throwaway env-gated generator (`GEN_COVERAGE=1 npx vitest run`
over a temp file importing `cardIntent` + `CARDS`); it's not part of the standing suite. Filter
[`handler-coverage.data.json`](handler-coverage.data.json) by `bucket` / `mechanics` /
`handCoded` to get the exact card list for any fix. Implementing these (the **fix campaign**) is
the next, separate effort ‚ÄĒ start at #1 (activated abilities) for the most cards-per-hour.
